* Introduction

The central problem this module solves: /How do you enforce that a plugin
implements the methods a benchmark expects, without waiting until the benchmark
tries to call them?/

Brain-Score's current answer is: you don't. =BrainModel= is an ABC, but nothing
checks compliance at registration time. A model that's missing =start_recording()=
enters the registry just fine, and fails only when a neural benchmark calls
=candidate.start_recording(...)= — deep inside a Jenkins scoring pipeline, minutes
after submission.

DMT's answer at the Blue Brain Project was a custom metaclass, =InterfaceMeta=,
that scanned class bodies for non-dunder methods and stored them as
=__requiredmethods__=. This worked but had two problems:

1. *Shared registry bug.* The =__implementation_registry__= was a mutable list on
   the =Interface= base class. Every subclass (=BrainModelInterface=,
   =BenchmarkInterface=) /shared/ the same list. Registering a model also appeared
   in the benchmark registry.

2. *Metaclass MRO conflicts.* Mixing =InterfaceMeta= with =ABCMeta= or PyTorch's
   module metaclass required a compound metaclass (=AIMeta=) — brittle and
   unintuitive.

This module replaces the metaclass with =__init_subclass__=, a hook introduced in
Python 3.6 ([[https://peps.python.org/pep-0487/][PEP 487]]). The idea is simple:
when you subclass =Interface=, Python calls =__init_subclass__= on the parent. We
use this hook to:

- Scan the new class body for non-dunder, non-private callables → these become
  =__required_methods__=
- Create a fresh =__implementation_registry__= dict for the new class (not shared
  with parent or siblings)

The result is the same interface enforcement, without a metaclass.

* Source

#+begin_src python
"""
Interface declaration using __init_subclass__ instead of metaclasses.

Inspired by DMT's InterfaceMeta, but avoids MRO conflicts and fixes
the shared-registry bug. Any non-dunder method in an Interface subclass
body becomes a required method. Implementations are validated at
registration time, not at runtime.

DMT (Blue Brain Project) used a custom metaclass (InterfaceMeta) that:
- Scanned the class body for non-dunder methods → __requiredmethods__
- Maintained __implementation_registry__ (but as a shared mutable list — bug)
- Validated implementations only up to the first missing method — bug

This version:
- Uses __init_subclass__ (no metaclass, no MRO conflicts)
- Each Interface subclass gets its own registry
- validate() reports ALL missing methods
"""

from __future__ import annotations

from typing import ClassVar


class Interface:
    """Base class for declaring structural interfaces.

    Subclass this to define a contract. Every non-dunder, non-private
    callable in the subclass body becomes a required method::

        class BrainModelInterface(Interface):
            def look_at(self, stimuli): ...
            def start_recording(self, target, time_bins): ...

    Then validate implementations::

        missing = BrainModelInterface.validate(MyModel)
        # [] if compliant, ['look_at', ...] if not
    """

    __required_methods__: ClassVar[set[str]] = set()
    __implementation_registry__: ClassVar[dict[str, type]] = {}

    def __init_subclass__(cls, **kwargs):
        super().__init_subclass__(**kwargs)
        # Capture all non-dunder, non-private callables as required methods.
        # Properties count — they define the interface contract too.
        cls.__required_methods__ = {
            name for name, val in vars(cls).items()
            if not name.startswith('_')
            and (callable(val) or isinstance(val, (property, classmethod, staticmethod)))
        }
        # Each subclass gets its own registry — not shared with parent.
        cls.__implementation_registry__ = {}

    @classmethod
    def validate(cls, impl) -> list[str]:
        """Return list of method names required by this Interface
        but missing from impl (a class or instance).

        Returns an empty list if impl is fully compliant.
        """
        return sorted([
            m for m in cls.__required_methods__
            if not hasattr(impl, m)
        ])

    @classmethod
    def is_implemented_by(cls, impl) -> bool:
        """True if impl has all required methods."""
        return len(cls.validate(impl)) == 0

    @classmethod
    def register(cls, name: str, impl_class: type):
        """Validate impl_class and register it under name.

        Raises TypeError if impl_class is missing required methods.
        """
        missing = cls.validate(impl_class)
        if missing:
            raise TypeError(
                f"{impl_class.__name__} does not implement {cls.__name__}. "
                f"Missing: {missing}"
            )
        cls.__implementation_registry__[name] = impl_class

    @classmethod
    def implementations(cls) -> dict[str, type]:
        """Return all registered implementations."""
        return dict(cls.__implementation_registry__)

    @classmethod
    def implementation_guide(cls) -> str:
        """Human-readable guide: what methods must an implementation provide?"""
        lines = [f"To implement {cls.__name__}, provide these methods:"]
        for method_name in sorted(cls.__required_methods__):
            method = getattr(cls, method_name, None)
            doc = getattr(method, '__doc__', None) or '(no docstring)'
            lines.append(f"  - {method_name}: {doc.strip()}")
        return '\n'.join(lines)
#+end_src

* Commentary

** The =__init_subclass__= Hook

The entire mechanism lives in one method:

#+begin_example python
def __init_subclass__(cls, **kwargs):
    super().__init_subclass__(**kwargs)
    cls.__required_methods__ = {
        name for name, val in vars(cls).items()
        if not name.startswith('_')
        and (callable(val) or isinstance(val, (property, classmethod, staticmethod)))
    }
    cls.__implementation_registry__ = {}
#+end_example

When you write:

#+begin_example python
class BrainModelInterface(Interface):
    def look_at(self, stimuli): ...
    def start_recording(self, target, time_bins): ...
#+end_example

Python calls =Interface.__init_subclass__(BrainModelInterface)=. At that point,
=vars(BrainModelInterface)= contains ={'look_at': <function>, 'start_recording':
<function>}=. We filter for non-dunder, non-private names → ={'look_at',
'start_recording'}=.

The key line is =cls.__implementation_registry__ = {}=. This creates a /new/ dict
on the subclass itself, shadowing the parent's class attribute. Without this,
=BrainModelInterface= and =BenchmarkInterface= would share
=Interface.__implementation_registry__= — exactly the bug in DMT's original
metaclass.

** Why =vars(cls)= Instead of =dir(cls)=

=vars(cls)= returns only the attributes defined /in this class body/, not
inherited ones. =dir(cls)= would include inherited methods from =Interface=
itself (=validate=, =register=, etc.), which we don't want as required methods.

This is a deliberate design choice: the required methods of an interface are
exactly what you write in its body.

** =validate()=: Report All Missing Methods

#+begin_example python
@classmethod
def validate(cls, impl) -> list[str]:
    return sorted([
        m for m in cls.__required_methods__
        if not hasattr(impl, m)
    ])
#+end_example

Returns a sorted list of /all/ missing methods, not just the first one. DMT's
original =validate()= returned after the first failure, leading to iterative
debugging: fix =look_at=, re-run, discover =start_recording= is also missing.

The choice to return a list (not raise) is deliberate: callers choose how to
handle non-compliance. =register()= raises; =is_implemented_by()= returns a bool;
external code can log warnings.

** =register()=: Validate-Then-Store

#+begin_example python
@classmethod
def register(cls, name: str, impl_class: type):
    missing = cls.validate(impl_class)
    if missing:
        raise TypeError(...)
    cls.__implementation_registry__[name] = impl_class
#+end_example

The invariant: /everything in the registry satisfies the interface/. This is the
"fail fast" principle — a =TypeError= at registration time is worth a thousand
=AttributeError= exceptions at scoring time.

** =implementation_guide()=: Self-Documentation

A convenience method that prints what's required:

#+begin_example
To implement BrainModelInterface, provide these methods:
  - identifier: Unique model name, e.g. 'alexnet'.
  - look_at: Present stimuli, return NeuroidAssembly or BehavioralAssembly.
  - start_recording: Configure the model to record from a brain region.
  - start_task: Configure the model for a behavioral task.
  - visual_degrees: How many visual degrees the model's input subtends.
#+end_example

This is included in =TypeError= messages from =@implements= (see [[file:04-adapter.org][WP-04]]),
giving the implementor a clear recipe for compliance.

** Properties as Interface Members

The comprehension includes =isinstance(val, (property, classmethod, staticmethod))=.
This is important because =BrainModelInterface.identifier= is a property, not a
method. Without this, the interface wouldn't require =identifier=, and models
missing it would pass validation.
